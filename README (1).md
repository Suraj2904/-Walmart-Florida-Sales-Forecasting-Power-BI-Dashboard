# 🛒 Walmart Florida — Sales Forecasting & Power BI Dashboard

An end-to-end data science project covering data ingestion, cleaning, EDA, feature engineering, ML forecasting (XGBoost + Prophet), and an interactive Power BI dashboard — all built on a synthetic Walmart Florida dataset.

---

## 📌 Project Overview

| Item | Detail |
|---|---|
| **Dataset** | Walmart Florida stores — September 2024 |
| **Scope** | Transactions, store locations, product catalog, distribution centers, inventory |
| **Models** | XGBoost (store × category level) + Prophet (chain-wide daily) |
| **Output** | 6 Power BI-ready CSV exports + `.pbix` dashboard |

## 🔄 Pipeline Walkthrough

### `walmart_full_pipeline.ipynb`

| Step | What it does |
|---|---|
| **1 — Data Loading** | Reads 4 CSVs + 2 SQLite tables |
| **2 — Data Cleaning** | Normalizes store IDs, fixes category names, strips `$` from prices, parses timestamps |
| **3 — Merging** | Joins transactions → stores → products; adds `margin` and `margin_pct` columns |
| **4 — EDA** | Daily trend, category revenue, regional breakdown, hour × weekday heatmap, top-10 stores |
| **5 — Feature Engineering** | Aggregates to daily (store × category); adds time features, lag features (`lag_1d`, `lag_7d`, `roll_3d`), and label encodings |
| **6 — Prophet Model** | Chain-wide daily revenue forecast with weekly seasonality; 14-day future projection |
| **7 — XGBoost Model** | Store × category granular forecast with 13 engineered features; feature importance plot |
| **8 — Model Comparison** | MAE / RMSE / MAPE side-by-side comparison |
| **9 — Business Insights** | Best categories, regions, days of week |
| **10 — Save Models** | Pickles XGBoost + both label encoders to `models/` |
| **11 — Store Map** | Interactive Plotly mapbox — store revenue by geography |

### `export_for_powerbi.py`

Loads the notebook's saved outputs (no re-processing) and generates 6 clean CSVs for Power BI:

| Export File | Used For |
|---|---|
| `walmart_sales.csv` | Revenue Overview, EDA pages |
| `store_locations.csv` | Store Map page |
| `daily_aggregated.csv` | Daily trend charts |
| `xgboost_predictions.csv` | Forecast Accuracy page |
| `prophet_forecast.csv` | Revenue Forecast page |
| `model_metrics.csv` | KPI cards, Model Comparison |

## 📊 Power BI Dashboard

The `.pbix` file connects to all 6 exports and includes:

- **Revenue Overview** — KPIs, category breakdown, regional performance
- **EDA Page** — Heatmaps, hourly/daily trends, margin analysis
- **Store Map** — Geographic bubble map of store-level revenue
- **Forecast Page** — Prophet 14-day forward projection with confidence intervals
- **Forecast Accuracy** — XGBoost actual vs predicted per store/category
- **Model Comparison** — MAE, RMSE, MAPE, and accuracy % for both models

**Relationships in Power BI:**
```
walmart_sales.store_id       → store_locations.store_id
walmart_sales.date           → daily_aggregated.date
xgboost_predictions.store_id → store_locations.store_id
xgboost_predictions.date     → daily_aggregated.date
```

## ⚙️ Setup & Usage

### 1. Install dependencies

```bash
pip install pandas numpy scikit-learn xgboost prophet joblib plotly matplotlib seaborn
```

### 2. Add raw data

Place raw files in the `raw/` directory (see structure above).

### 3. Run the notebook

```bash
jupyter notebook walmart_full_pipeline.ipynb
```

Run all cells top to bottom. This saves processed data to `data/processed/` and models to `models/`.

### 4. Export for Power BI

```bash
python export_for_powerbi.py
```

This generates all 6 CSVs in `powerbi_exports/`.

### 5. Open the dashboard

Open `Walmart_Sales_Dashboard.pbix` in Power BI Desktop and point the data sources to your `powerbi_exports/` folder.
